# Directive: Create Professional Landing Page

## Goal
Create a high-quality, professional, and minimalistic landing page for the "Dental Clinic Automation" project. The page serves as a portfolio piece/sales page for the user (The "Expert") to send to potential clients (Dental Clinic Owners).

## Requirements

### 1. Content Strategy
- **Persona**: The user is a "Systems & Workflow Specialist" or "Automation Architect", not just an "AI Developer".
- **Tone**: Professional, confident, results-oriented, minimalistic. Avoid technical jargon like "Python scripts", "n8n nodes" in the main copy. Use business outcome terms ("Reducing no-shows", "Seemless patient intake").
- **Sections**:
    1.  **Hero**: Strong headline, value proposition, "View Demo" button.
    2.  **About**: Brief "Who am I". Focus on competence (e.g., "I bridge the gap between clinical needs and digital efficiency").
    3.  **The Project**: Brief description of the Dental Automation System (Lead Intake, Reminders, Reviews).
    4.  **Footer**: Signature and clear contact method.

### 2. Design & Aesthetics
- **Style**: Ultra-clean, premium, "Medical Tech".
- **Color Palette**: 
    - Text: Dark Slate / Charcoal (Soft Black).
    - Accents: Clinical Blue/Teal (e.g., `#0ea5e9` or `#0d9488`) but used sparingly.
    - Background: Clean white/off-white with subtle gradients or "glass" cards for depth.
- **Typography**: Sans-serif, modern (Inter or Outfit). Good leading and tracking.
- **Interactions**: Subtle hover states, smooth scrolling, fade-in animations on load.

### 3. Technical Implementation
- **Stack**: Pure HTML5, CSS3 (Vanilla). No frameworks needed for a single page.
- **File Structure**:
    - `landing_page/index.html`
    - `landing_page/style.css`
- **Responsive**: Must look perfect on mobile.

## Execution Steps
1.  Setup the folder structure.
2.  Write the HTML with semantic structure.
3.  Apply CSS variables for consistency.
4.  Implement the "Wow" factor (animations/glassmorphism).
5.  Validate links (Demo link can be `#demo` or a placeholder).
